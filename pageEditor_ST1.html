2. Shared Template & Global CSS

HTML Structure

The shared template provides a consistent <nav>, <main>, and <footer> structure used across all team pages. The <nav> element uses a flexbox layout to align the logo and navigation links. The <footer> uses CSS Grid with auto-fit to create a responsive three-column layout that collapses gracefully on mobile.

CSS Techniques

The global CSS (style.css) uses CSS custom properties (variables) to define the shared colour palette consistently across all pages. This ensures that any future palette change requires only a single edit. Flexbox is used for the navbar and footer. No grid frameworks were used, all layout is hand-coded.

Key decisions: the gradient navbar with a gradient from #4facfe to #00f2fe was chosen to evoke ocean water and distinguish the navigation visually from the deep-ocean body background. Link hover states use a scale transform and a box-shadow to provide clear affordance without disrupting layout.

3. Home Page

HTML Structure

The Home Page uses <section> elements to semantically group the hero area, the three card-based content sections, and the interactive gallery of content pages. The hero uses a linear gradient overlay to ensure text legibility across all screen sizes.

CSS Techniques

Five embedded CSS rules handle page-specific styles: card hover elevation (translateY(-10px)), image zoom on hover (scale), responsive grid for the content gallery, interactive gallery tile hover state, and mobile typography adjustments via a media query at 600px.

The content gallery tiles use background-image with a gradient overlay for text legibility, combined with a cubic-bezier transition for a polished spring-like scaling effect on hover.

Design Rationale

Cards were chosen as the primary layout pattern as they work well with the navy-on-deep-ocean colour palette and allow images, headings, and bullet points to coexist in a scannable format. The three cards directly address the brief's requirement for original, action-focused SDG content rather than a description of the UN goal.

4. Gallery Page

HTML Structure

The gallery is dynamically built using JavaScript, injecting <article> elements into a CSS Grid container. Each article has appropriate role="listitem", tabindex="0", and aria-label attributes. The modal uses role="dialog" and aria-modal="true".

JavaScript Interactivity

All interactivity is JavaScript-driven as required. Hover state is applied via mouseenter event listeners. Click-to-expand opens a modal populated dynamically from the gallery data array. Close is supported via a button click, Escape key, and click-outside-modal. Font and colour customisation within the modal are applied by JavaScript by modifying element.style and className properties.

5. Content Page - The Ocean Crisis

HTML Structure

The content page uses a two-column CSS Grid layout with a sticky side navigation panel and a <article> main content area. Heading hierarchy follows <h1> <h2> <h3> strictly. All four images include meaningful alt text. Internal anchor links in the side nav target section IDs.

CSS Techniques

position: sticky on the side nav keeps it visible as the user scrolls, improving navigation of the long-form content. Images use object-fit: cover to maintain consistent height without distortion. A "Go to Top" button uses position: fixed to remain visible at all times.

6. Challenges & Lessons Learned

Challenge 1: Navbar width on different screens. The fixed navbar initially had a hardcoded pixel width that broke on larger monitors. Resolved by switching to width: 100vw and removing the fixed pixel value.

Challenge 2: Gallery modal focus management. After opening the modal, keyboard focus remained on the triggering card, making it difficult to navigate the modal. Resolved by explicitly calling closeBtn.focus() on modal open.

Challenge 3: CSS specificity conflicts. Page-specific embedded CSS initially overrode global heading styles due to specificity. Resolved by scoping embedded rules to page-specific class selectors rather than bare element selectors.

Lesson: Establishing a clear naming convention and commenting all shared CSS from the start of the project would have prevented several merge conflicts when integrating team members' pages.

7. Accessibility Considerations

All images across my implemented pages include descriptive alt text that conveys the informational content of the image to screen-reader users. Decorative images use empty alt text.

Semantic HTML elements (<nav>, <main>, <footer>, <article>, <section>) are used throughout to provide meaningful structure. This allows assistive technologies to navigate the page by landmarks.

All interactive elements (gallery cards, modal close button, navigation links) are keyboard-accessible. The gallery cards use tabindex="0" and respond to Enter/Space keydown events. The modal traps focus within it while open.

Colour contrast ratios between text and background meet WCAG 2.1 AA standards. The cyan text on navy background achieves a ratio above 4.5:1.
