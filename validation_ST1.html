Reflection on Validation

The W3C validation process was a valuable part of the development workflow. During initial validation, the home.html file contained an unclosed section tag inside the hero area, which was caught immediately and corrected. A stray div in gallery.html was also identified this way.

Running validation iteratively - after each significant code change rather than only at the end - made it significantly easier to isolate the source of errors. This is a practice I will carry into all future web development work.

No pages in my submission produced validation errors in their final submitted state. Minor informational warnings (such as the type attribute on JavaScript script elements being unnecessary in HTML 5) were acknowledged but do not constitute errors.
