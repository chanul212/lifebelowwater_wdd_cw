2. User Profile Page

Technical Description

The User Profile page uses a two-column CSS Grid layout (grid-template-columns: 1fr 1fr) to place the step-based input panel alongside the live profile preview card. On screens below 768px a media query collapses this to a single column.

The multi-step interface is controlled by JavaScript. Each step corresponds to a <div class="step-content"> block. Only the active step's block is shown (display: block), all others are hidden. Step tab buttons at the top of the panel are styled with .active and .done CSS classes, applied and removed by JavaScript as the user progresses. This avoids full page reloads between steps.

The live preview card is updated in real time by JavaScript input and change event listeners on each form field. When the user types their name, the preview card's name element updates immediately via element.textContent. Avatar selection uses a radio-button-like approach where clicking an avatar image calls a JavaScript function that updates the preview card's <img> src and adds a selected border to the chosen option. Pledge checkboxes build a displayed list in the preview card, appending or removing <li> elements via JavaScript DOM manipulation.

Accessibility

Step tab buttons are native <button> elements, keyboard-accessible by default. Each input field has an associated <label>. The two-column grid collapses on mobile so the input and preview are stacked vertically and readable without horizontal scrolling. The profile layout uses aria-label on the steps panel for screen-reader landmark clarity.

3. Sitemap

Technical Description

The sitemap is implemented as an inline SVG with a viewBox="0 0 960 620" coordinate system. Setting width: 100%; height: auto on the SVG element makes it fully responsive -the diagram scales to fit any container without losing its internal proportions or requiring JavaScript resize handlers.

The site hierarchy is represented using SVG <rect> elements for nodes and <line> elements for connectors. Each node is wrapped in an <a> element linking to the corresponding HTML page, making the entire diagram interactive. Nodes use CSS class .site-node with a CSS transition on fill so hover states animate smoothly. The current page (Sitemap) is highlighted with the cyan fill colour and a gold star label to distinguish it visually.

Text labels inside the SVG use <text> elements with text-anchor="middle" and pointer-events: none to prevent text from intercepting mouse clicks that should reach the underlying <rect>. A legend below the SVG uses HTML flexbox to display colour swatches explaining the node colour coding (Home, Main Pages, Content Pages, Student 4 pages).

Accessibility

The SVG includes a <title> and <desc> element describing the diagram for screen readers. Each <a> node wrapper includes an aria-label describing the destination page. All interactive nodes have tabindex="0", allowing keyboard navigation through every node in the diagram.

4. Content Page - Sustainable Fisheries & Blue Economy

Technical Description

The content page uses the same shared two-column CSS Grid layout as the other content pages: a sticky 240px side navigation column and a 1fr main article column. CSS custom property variables (--teal, --cyan, --seafoam) are used consistently throughout for colours, ensuring visual alignment with the rest of the site without duplicating hex values.

The page covers the overfishing crisis, blue economy framework, MSC certification, and a Sri Lanka fisheries focus section. Stat callout boxes use the shared .stat-box class pattern with a large .stat-num element in seafoam. Images are styled with object-fit: cover; height: 300px and a subtle teal border to frame them consistently. Gold border styling (border: 2px solid var(--seafoam)) is used on Student 4's assigned elements to visually distinguish them on the sitemap, mirroring the gold star convention in the SVG diagram.

Accessibility

The side navigation uses aria-label="Page sections". All images have descriptive alt text. Internal anchor links use the shared scroll-behavior: smooth. The layout collapses to a single column on mobile and the side nav is hidden below 768px.

5. Challenges & Lessons Learned

Challenge 1: SVG text clipping on small screens. At narrow viewport widths, SVG text labels inside node rectangles were clipped because the SVG scaled proportionally but the text font-size did not. Since SVG text does not respond to CSS clamp(), the solution was to wrap the entire SVG in a horizontally scrollable container (overflow-x: auto) with a minimum width, so the diagram always renders at a legible size.

Challenge 2: Multi-step profile state persistence. When the user navigated back to a completed step, the previously entered values disappeared from the input fields because step visibility was toggled by showing/hiding divs. Resolved by reading stored values from a JavaScript state object when switching to a step, re-populating inputs via JavaScript before making the step visible.

Challenge 3: Live preview card not updating on tab-away. The preview card failed to update when users changed a select element without using the mouse. Resolved by adding both input and change event listeners to each field to cover all interaction methods.

Lesson: SVG is powerful for diagrams but requires careful planning for responsive behaviour from the start, as retrofitting responsiveness to a large fixed-coordinate SVG is time-consuming. Designing with a scalable viewBox approach from the outset avoids this.

6. Accessibility Considerations

All images across my pages include descriptive alt text. The SVG sitemap includes a <title> and <desc> for screen readers, and all interactive nodes have aria-label attributes. Form inputs on the profile page have associated labels. The multi-step interface uses native <button> elements for step navigation. Keyboard focus is maintained correctly across step transitions. Colour contrast on all text elements meets WCAG 2.1 AA standards.
