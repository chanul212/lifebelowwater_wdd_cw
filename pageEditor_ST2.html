2. Splash Screen

Technical Description

The splash screen uses a full-viewport CSS layout with height: 100vh and overflow: hidden to fill the screen without scrolling. The animated wave background is created entirely with CSS using ::before and ::after pseudo-elements on a .wave-bg div, each animated with a custom @keyframes wave rule that applies a rotating translateX transform to two overlapping ellipses of different opacity and animation duration, creating the illusion of layered ocean waves.

The content fades in on load using a @keyframes fadeIn animation applied to the .splash-content wrapper, transitioning from opacity: 0; translateY(30px) to fully visible. This is purely CSS-driven with no JavaScript required for the animation itself.

A JavaScript countdown timer uses setInterval to decrement a seconds counter every 1000ms, updating the #countdown element's text content and redirecting via window.location.href when it reaches zero. The skip button's click listener calls clearInterval on the running timer before redirecting, preventing a double-redirect race condition. A <meta http-equiv="refresh"> tag provides a fallback redirect for non-JavaScript environments.

Accessibility

The loading animation div includes aria-label="Loading animation" so screen readers announce it meaningfully. The skip button is a native <a> element, making it keyboard-focusable by default. The SDG badge in the corner uses a fixed position but does not obscure any interactive content.

3. Quiz / Action Impact Simulator

Technical Description

The Action Impact Simulator presents a grid of clickable action cards, each representing a real-world SDG 14 action (e.g. reducing plastic use, buying sustainable seafood). The grid uses display: grid with repeat(auto-fit, minmax(260px, 1fr)) to adapt the number of columns responsively without any media queries.

Each card stores a numeric impact score in its JavaScript data object. When clicked, JavaScript toggles a .selected class on the card element and adds or subtracts the card's impact value from a running total. The .selected state is styled with a cyan border and a CSS ::after pseudo-element displaying a checkmark, providing instant visual feedback without DOM injection.

The score display uses a CSS progress bar (#progress-bar) whose width is updated as a percentage via JavaScript each time a card is toggled. The feedback panel text, impact label colour, and message are dynamically updated based on score thresholds using a series of conditional checks. A reset button clears all selections, zeroes the score, and resets the progress bar width to 0% via JavaScript.

Accessibility

Each action card includes a descriptive aria-label. The progress bar uses role="progressbar" with aria-valuenow, aria-valuemin, and aria-valuemax attributes updated dynamically by JavaScript. The reset button is a native <button> element, keyboard accessible by default.

4. Content Page – Marine Biodiversity & Coral Reefs

Technical Description

The content page uses a two-column CSS Grid layout: a 240px sticky side navigation column and a flexible 1fr article column. The position: sticky; top: 100px on the side nav keeps it anchored in view as the user scrolls through the long-form article, significantly improving navigation of dense content without JavaScript.

The article uses a strict heading hierarchy: <h1> for the page title, <h2> for major sections (styled with a left teal border and the seafoam colour variable), and <h3> for subsections. All images use object-fit: cover with a fixed height of 300px to maintain visual consistency regardless of the source image's aspect ratio. Stat callout boxes use a semi-transparent navy background with a teal border, drawing the eye without interrupting reading flow.

A fixed "Go to Top" button uses position: fixed; bottom: 30px; right: 30px and links to #top on the <main> element, providing persistent navigation for long pages.

Accessibility

The side navigation uses aria-label="Page sections" to distinguish it from the main site navigation. All images include descriptive alt text. The grid collapses to a single column on screens below 768px, and the side navigation is hidden on mobile via a media query to avoid layout clutter on small screens.

5. Challenges & Lessons Learned

Challenge 1: Countdown timer double-redirect. When the skip button was clicked just as the timer fired, both redirect paths executed simultaneously, causing unpredictable behaviour. Resolved by storing the setInterval return value and calling clearInterval inside the skip button's event listener before setting window.location.href.

Challenge 2: AIS score going negative. When users deselected all cards after previously selecting them, rounding errors in the percentage calculation caused the progress bar width to briefly go below 0%. Resolved by clamping the score to a minimum of 0 before calculating the percentage.

Challenge 3: CSS wave animation performance. The wave pseudo-elements initially caused visible jank on lower-powered devices due to layout-triggering properties. Resolved by replacing margin animations with transform-only animations, which are GPU-accelerated and do not trigger layout recalculations.

Lesson: Testing interactive features on multiple devices early -not just at the end -prevents integration surprises. JavaScript timer and animation interactions are particularly sensitive to device performance.

6. Accessibility Considerations

All images across my pages include descriptive alt text conveying the informational content of each image. Semantic HTML elements (<nav>, <main>, <article>, <section>) are used throughout. All interactive elements -action cards, reset button, skip button -are keyboard accessible, either by being native elements or by having tabindex="0" and keydown event listeners.

Colour contrast between the cyan text and navy/dark backgrounds meets WCAG 2.1 AA requirements. The splash screen's animated wave background does not contain any information and is purely decorative, so it does not require an alternative for screen-reader users.
